# 多模态识别任务管理器

## 项目概述

黑客松48小时项目。

**解决问题**：借用AI，从多种格式文件中提取有关个人时间安排的信息，并给出可视化的任务安排界面。

包括语音输入、文字输入、图片海报、聊天记录等。

**目标平台**：Windows Demo。

**技术路线**：不采用单一多模态模型解决所有问题，而是通过文件后缀将不同格式分发给专门解析模型，分别解析出文字后统一交给一个LLM生成结构化任务信息，最后输出可视化结果。

## 逐组件分析

### 图片 OCR（.jpg / .png / 海报）

使用 GPT-4o / Claude 原生视觉能力，无需单独 OCR 服务。海报、截图、聊天记录截图都能直接丢进去出文字。

```python
# 集成难度：★☆☆☆☆
import base64
from openai import OpenAI

def ocr_image(file_path):
    client = OpenAI()
    with open(file_path, "rb") as f:
        b64 = base64.b64encode(f.read()).decode()
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{
            "role": "user",
            "content": [
                {"type": "text", "text": "提取图片中的所有文字，保持原有格式"},
                {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{b64}"}}
            ]
        }]
    )
    return response.choices[0].message.content
```

### 音频 ASR（.mp3 / .wav / .m4a）

使用 OpenAI Whisper API，支持中文，一个 HTTP 请求即可。注意大文件需要先分段（25MB 限制）。

```python
# 集成难度：★☆☆☆☆
from openai import OpenAI

def transcribe_audio(file_path):
    client = OpenAI()
    with open(file_path, "rb") as f:
        transcript = client.audio.transcriptions.create(
            model="whisper-1",
            file=f,
            language="zh"
        )
    return transcript.text
```

### 文档解析（.docx / .pdf / .txt）

纯文本提取，不涉及 AI 推理，轻量 Python 库搞定。

```python
# 集成难度：★★☆☆☆
import docx        # python-docx
import pymupdf     # PyMuPDF

def parse_document(file_path):
    if file_path.endswith('.docx'):
        doc = docx.Document(file_path)
        return '\n'.join([p.text for p in doc.paragraphs])
    elif file_path.endswith('.pdf'):
        doc = pymupdf.open(file_path)
        return '\n'.join([page.get_text() for page in doc])
    elif file_path.endswith('.txt'):
        with open(file_path, 'r', encoding='utf-8') as f:
            return f.read()
    raise ValueError(f"不支持的文件格式: {file_path}")
```

### 聊天记录

直接文本输入，无需解析，原样送入 LLM。

### 统一 LLM 提取层（核心）

所有解析器产出纯文本后，统一交给一个 LLM 做结构化提取。这是经多个黑客松项目验证可行的架构模式。

```
你是一个个人时间安排提取助手。从以下文本中提取所有任务/事件/日程信息，输出JSON。

输入文本可能来自：
- 对话/聊天记录
- 海报/图片中的文字
- 语音转录文本
- 文档/通知

要求输出格式：
[{
  "title": "任务名称",
  "datetime": "2026-07-16T14:00:00",
  "location": "地点（如有）",
  "attendees": ["参与者"],
  "notes": "备注",
  "confidence": 0.85,
  "source": "来源文件名"
}]

置信度评分标准：
- 0.9-1.0：时间、地点、事件都明确
- 0.7-0.9：事件明确但时间/地点有歧义（如"下周"、"到时候"）
- 0.5-0.7：仅能推断出模糊意图
- <0.5：无法提取有效事件，不输出
```

### 置信度路由

```
≥ 80% → 自动添加到任务列表
50-79% → 展示给用户确认（YES/NO）
< 50% → 丢弃
```

## 可行性结论

**可行 ✅**。该方案与常见误区有本质区别：

- ❌ 误区：离线 Whisper + Tesseract + CLIP + Mistral 全本地部署 → 依赖地狱、内存爆炸、Windows 不兼容
- ✅ 本方案：云端 API 各司其职 → 纯文本汇聚 → 统一 LLM → **薄封装、低耦合、平台无关**

架构清晰，每个组件独立可测试，API 集成均是"一个 HTTP 请求搞定"的级别。
