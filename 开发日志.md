### 技术选型确认

| 组件 | 选型 | 理由 |
|------|------|------|
| 后端框架 | FastAPI | 异步支持好，AI 工具链自然语言 |
| LLM API | OpenAI SDK | 兼容公司 API 代理 |
| ASR | Whisper API | 一个 HTTP 请求，无需本地模型 |
| OCR | GPT-4o Vision | 直接传图片，无需独立 OCR 服务 |
| 前端 | React + TypeScript + Vite | 开发快，生态成熟 |
| 日历 | FullCalendar v6 | 周/月/列表三视图，中文支持 |
| 样式 | Tailwind CSS v4 | Vite 插件，零配置 |

### 后端搭建

- FastAPI 项目骨架（`main.py`、`config.py`）
- 数据模型：`Task`、`ExtractionResult`、`UploadResponse`
- 文件调度器 `Dispatcher` — 按后缀分发给对应解析器
- 5 个解析器：
  - `TextParser` — `.txt`/`.md`/`.csv` 直接读取
  - `DocxParser` — `python-docx` 提取段落
  - `PdfParser` — `PyMuPDF` 逐页提取
  - `ImageParser` — GPT-4o Vision API OCR
  - `AudioParser` — Whisper API 语音转文字
- LLM 提取层：中文 Prompt 工程 + JSON Schema Structured Outputs
- 置信度路由：≥80% 自动添加，50-79% 待确认，<50% 丢弃
- 内存任务存储：CRUD + Demo 数据加载

### 前端搭建

- React + TypeScript + Vite 项目初始化
- 组件树：`Layout → InputPanel → (TextInput | FileUpload) + TaskTimeline`
- FullCalendar 三视图：周视图 / 月视图 / 列表视图
- 拖拽文件上传（react-dropzone）
- 待确认任务浮动按钮 + 确认弹窗（PendingBadge）
- Vite 代理配置 `/api → localhost:8000`

### Demo 数据

7 条覆盖一周的日程任务：会议、健身、医疗、社交、截止日期、户外活动

## 2026-07-14 集成调试

### Bug 修复记录

#### 1. PyMuPDF 1.24.0 编译失败

**现象：** `pip install pymupdf==1.24.0` 报错 `Unable to find Visual Studio`

**原因：** 1.24.0 在 Windows Python 3.13 上没有预编译 wheel，需要本地编译 C 扩展，而本机未安装 Visual Studio

**修复：** 升级到 `pymupdf>=1.28.0`，该版本提供了 `cp310-abi3-win_amd64` 预编译 wheel

**耗时：** 10 分钟

---

#### 2. image_parser / audio_parser import 路径错误

**现象：**
```
ModuleNotFoundError: No module named 'backend.services.config'
```

**原因：** `image_parser.py` 和 `audio_parser.py` 使用 `from ..config import settings`，但由于它们在 `services/parsers/` 目录下（深一层），需要三个点 `...` 才能回到 `backend` 包的根

**修复：** `from ..config import settings` → `from ...config import settings`

**耗时：** 2 分钟

---

#### 3. useTasks.ts JSX 语法错误

**现象：** Vite 编译报错 `Expected ">" but found "value"` 在 `useTasks.ts:150`

**原因：** 文件包含 JSX（`<TaskContext.Provider value={...}>`），但扩展名是 `.ts`，esbuild 不会用 JSX 模式解析 `.ts` 文件

**修复：** 重命名为 `useTasks.tsx`

**耗时：** 2 分钟

---

#### 4. FullCalendar 视图切换无效

**现象：** 点击「周」「月」「列表」按钮，视图不切换

**原因：** FullCalendar 的 `initialView` prop 仅在首次挂载时生效，后续 state 变化不会触发视图更新

**修复：** 给 `<FullCalendar>` 组件添加 `key={view}`，每次视图变化时强制 React 重新挂载组件

**耗时：** 2 分钟

---

#### 5. httpx 版本冲突

**现象：** 
```
AsyncClient.__init__() got an unexpected keyword argument 'proxies'
```

**原因：** `httpx 0.28.x` 移除了 `proxies` 参数，而 `openai 1.55.0` 仍然传递该参数

**修复：** 升级 `openai` 到 `2.45.0`（最新版本已适配新版 httpx）

**耗时：** 5 分钟

---

#### 6. 模型兼容性

**现象：** `.env` 中 `MODEL_NAME=gpt-image-2`，调用 chat completions 报错：
```
The 'gpt-image-2' model is not supported when using Codex with a ChatGPT account
```

**原因：** `gpt-image-2` 是图像生成模型，不支持 chat completions

**修复：** 改为 `gpt-5.5`（公司 API 代理支持的文本模型）

**耗时：** 5 分钟
